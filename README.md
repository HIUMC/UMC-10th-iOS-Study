# 🍏 UMC iOS 파트 스터디

한 학기 동안 진행되는 iOS 파트 스터디 레포지토리입니다.
원활한 코드 공유와 코드 리뷰를 위해 아래의 Git 컨벤션을 준수해 주세요.

## 🌿 Git Workflow (브랜치 전략)

우리는 하나의 저장소(Repository)에서 각자의 브랜치를 통해 과제를 진행합니다.

1. **브랜치 생성:** `main` 브랜치에서 자신의 닉네임으로 된 개인 브랜치를 생성합니다.
   - 브랜치명 규칙: `닉네임` (예: `leehan`)
2. **과제 진행:** 매주 과제는 본인의 브랜치에서 작업합니다.
3. **PR 생성:** 매주 스터디 모임 전, 본인의 닉네임 브랜치에서 `main` 브랜치로 Pull Request를 생성합니다.
4. **리뷰 및 머지:** 스터디 시간에 다 같이 코드를 리뷰한 후, 파트장이 `main` 브랜치로 머지(Merge)합니다.

## 📝 Commit Convention

커밋 메시지는 팀원들이 작업 내역을 쉽게 파악할 수 있도록 아래 규칙을 따릅니다.

### 💡 커밋 메시지 구조
`태그: 작업 내용`

### 🏷 태그 종류
* `feat` : 새로운 기능 추가, UI 구현
* `fix` : 버그 수정, 에러 해결
* `design` : UI 디자인 변경 (기능 변경 없는 CSS, 레이아웃 수정)
* `refactor` : 코드 리팩토링 (기능 변화 없이 코드 구조 개선)
* `docs` : README 파일 등 문서 수정
* `chore` : 프로젝트 설정 변경, 패키지 매니저(SPM 등) 수정, 주석 추가 및 삭제

### ✅ 커밋 메시지 예시
* `feat: 1주차 메인 화면 뷰 구현`
* `fix: 리스트 스크롤 시 앱이 튕기는 현상 수정`
* `design: 폰트 사이즈 및 여백 조정`
* `chore: 필요 없는 print문 삭제 및 코드 정리`

## 🚀 Pull Request (PR) Convention

매주 과제가 완료되면 PR을 올려주세요. PR 템플릿에 맞춰 작성하면 리뷰가 훨씬 수월해집니다.

### 📌 PR 제목 규칙
* `[닉네임/이름] N주차 과제 제출`
* **예시:** `[리한/이한결] 1주차 과제 제출`
